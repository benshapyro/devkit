# Content Pillars & Topic Clusters

Strategy for building topical authority through interconnected content.

## The Pillar-Cluster Model

```
                    ┌─────────────────┐
                    │   PILLAR PAGE   │
                    │ (Comprehensive) │
                    └────────┬────────┘
                             │
           ┌─────────────────┼─────────────────┐
           │                 │                 │
     ┌─────┴─────┐     ┌─────┴─────┐     ┌─────┴─────┐
     │  CLUSTER  │     │  CLUSTER  │     │  CLUSTER  │
     │  CONTENT  │     │  CONTENT  │     │  CONTENT  │
     └───────────┘     └───────────┘     └───────────┘
           │                 │                 │
      (Long-tail)       (Long-tail)       (Long-tail)
       keywords          keywords          keywords
```

### How It Works

1. **Pillar Page:** Comprehensive guide on broad topic (2,000-5,000 words)
2. **Cluster Content:** In-depth posts on specific subtopics (1,000-2,000 words)
3. **Internal Links:** All cluster content links to pillar; pillar links to clusters
4. **SEO Benefit:** Establishes topical authority; passes link equity

## Defining Your Pillars

### Selection Criteria

| Criterion | Weight | Questions to Ask |
|-----------|--------|------------------|
| Business Relevance | High | Does this topic directly relate to what we sell? |
| Search Demand | High | Do people search for this topic? (Volume > 1,000/mo) |
| Expertise | Medium | Can we create authoritative content here? |
| Competition | Medium | Can we realistically rank? (Not dominated by giants) |
| Audience Fit | High | Does our target audience care about this? |

## Building Authority Over Time

### Phase 1: Foundation (Months 1-2)

```
Week 1-2: Publish pillar page
Week 3-4: Publish 2-3 high-priority cluster posts
Week 5-8: Publish 4-6 more cluster posts
         Add internal links between all content
         Begin link building to pillar
```

### Phase 2: Expansion (Months 3-4)

```
Continue cluster content (aim for 10-15 total per pillar)
Update pillar with new links
Create second pillar if resources allow
Begin content refresh cycle for existing posts
```

### Phase 3: Optimization (Months 5-6)

```
Analyze performance of all content
Refresh underperforming pieces
Expand successful cluster posts
Add new cluster content based on data
Optimize internal linking
```

### Success Indicators

| Metric | Months 1-2 | Months 3-4 | Months 5-6 |
|--------|------------|------------|------------|
| Pillar ranking | 20-50 | 10-20 | Top 10 |
| Cluster pieces | 5-8 | 10-15 | 15-20 |
| Organic traffic | Baseline | +50% | +100% |
| Backlinks (pillar) | 5-10 | 15-25 | 30+ |
| Featured snippets | 0-1 | 2-5 | 5-10 |

## Cross-Linking Best Practices

### Do

1. **Link from pillar to all clusters** in relevant sections
2. **Link from clusters back to pillar** (usually in intro or conclusion)
3. **Cross-link related clusters** where natural
4. **Use descriptive anchor text** (not "click here")
5. **Update old content** to link to new pieces
6. **Audit links quarterly** to fix broken links

### Don't

1. **Don't force links** where they don't fit naturally
2. **Don't use same anchor text** for every link to same page
3. **Don't link cluster to cluster** without also linking to pillar
4. **Don't over-link** (3-5 internal links per 1,000 words is sufficient)
5. **Don't forget to add links** when publishing new content
