## When to Use

User says something like:
- "Create a strategy deck for [client]"
- "Build a presentation of our findings"
- "Generate the strategy presentation"
- "Make slides for the readout meeting"

**Typical audience:** Leadership team, steering committee, key stakeholders
**Presentation time:** 30-60 minutes
**Slide count:** 15-25 slides

## Deck Structure

### Standard Strategy Deck (20 slides)

| Section | Slides | Purpose |
|---------|--------|---------|
| **Opening** | 1-3 | Set context, agenda, executive summary |
| **Current State** | 4-7 | Challenges, landscape, what we found |
| **Insights** | 8-11 | Patterns, key learnings, stakeholder themes |
| **Recommendations** | 12-16 | Prioritized solutions, quick wins, strategic bets |
| **Roadmap** | 17-19 | Phased plan, timeline, milestones |
| **Close** | 20 | Next steps, ask, discussion |

### Section Details

#### Section 1: Opening (Slides 1-3)

**Slide 1: Title Slide**
- Engagement name
- Client name
- Date
- "Strategy Recommendations" or similar

**Slide 2: Agenda**
- What we'll cover today
- Keep to 4-6 items max
- Set expectations for discussion time

**Slide 3: Executive Summary**
- 3-4 key findings (one line each)
- Primary recommendation
- Expected outcome
- This slide should standalone — executives may only see this

#### Section 2: Current State (Slides 4-7)

**Slide 4: Engagement Context**
- Why we're here (business driver)
- Scope of discovery
- Sessions conducted, stakeholders interviewed

**Slide 5: Challenge Landscape**
- Top challenges identified (from synthesis prioritizer)
- Organized by dimension or priority
- Impact indicators

**Slide 6-7: Key Findings by Dimension**
- People findings (stakeholder dynamics)
- Process findings (workflow pain points)
- Technology findings (system landscape)
- Use icons or visuals to break up text

#### Section 3: Insights (Slides 8-11)

**Slide 8: Pattern Summary**
- 3-4 dominant patterns (from synthesis)
- Evidence count for each
- Visual treatment (icons, color coding)

**Slide 9-10: Deep Dive on Top Patterns**
- One slide per critical pattern
- What we observed
- Supporting evidence (quotes, data)
- Implication for strategy

**Slide 11: Stakeholder Alignment**
- Key players and their positions
- Champions vs. skeptics
- Alignment risks to address

#### Section 4: Recommendations (Slides 12-16)

**Slide 12: Recommendation Overview**
- All recommendations at a glance
- Categorized: Quick Wins, Strategic, Future
- Priority ranking visible

**Slide 13: Quick Wins (Detail)**
- 2-3 immediate opportunities
- Impact vs. effort positioning
- Expected timeline and outcome

**Slide 14-15: Strategic Recommendations**
- One slide per major recommendation
- What, why, expected impact
- Dependencies and risks
- Connection to challenges

**Slide 16: Solutions Summary**
- How recommendations address challenges
- Visual mapping (challenge → solution)
- ROI indicators where available

#### Section 5: Roadmap (Slides 17-19)

**Slide 17: Phased Approach**
- Phase 1: Quick Wins (Weeks 1-4)
- Phase 2: Foundation (Months 2-3)
- Phase 3: Scale (Months 4-6)
- Visual timeline

**Slide 18: Phase 1 Detail**
- Specific initiatives
- Owners (suggested)
- Success metrics
- Dependencies

**Slide 19: Investment & Returns**
- Resource requirements
- Expected benefits
- Risk factors
- Comparison to status quo cost

#### Section 6: Close (Slide 20)

**Slide 20: Next Steps & Discussion**
- 3-5 immediate next steps
- Decisions needed
- Open questions for discussion
- Contact information

## Content Guidelines

### Writing for Slides

**Headlines:**
- Make headlines complete thoughts, not labels
- Bad: "Challenges"
- Good: "Manual processes consume 20+ hours per week"

**Bullet Points:**
- Maximum 5 bullets per slide
- Each bullet: 1-2 lines max
- Start with action verbs or key nouns
- Parallel structure across bullets

**Data Presentation:**
- Round numbers for impact (20+ hours, not 21.3 hours)
- Use comparisons (3x faster, 50% reduction)
- Cite sources (stakeholder quotes, session dates)

### Evidence Standards

| Confidence | How to Present |
|------------|----------------|
| High (direct quote) | "Stakeholder said: '[quote]'" |
| Medium (inference) | "Multiple stakeholders indicated..." |
| Low (interpretation) | "Initial signals suggest..." (flag for validation) |

### Avoiding Common Mistakes

| Mistake | Fix |
|---------|-----|
| Too much text | One idea per slide, use speaker notes |
| Jargon overload | Use client's terminology |
| No "so what" | Every finding needs an implication |
| Missing evidence | Support claims with discovery data |
| Generic recommendations | Be specific to this client's context |

## Visual Standards

### Cadre Brand Application

**Colors:**
- Primary backgrounds: White (#FFFFFF)
- Section dividers: Deep Blue (#034377)
- Headlines: Black (#000000) or Deep Blue
- Accent/highlights: Coral Red (#DB4545)
- Secondary accents: Medium Blue (#08749B), Teal (#0F8090)

**Typography:**
- Headlines: Inter Bold (or Helvetica Neue Bold)
- Body text: Inter Regular
- Minimum font size: 18pt for body, 24pt for headlines

**Spacing:**
- Generous margins (0.5" minimum)
- Consistent padding between elements
- White space is intentional — don't fill every inch

**Charts and Graphics:**
- Use Cadre color palette
- Simple, clean visualizations
- Label directly (avoid legends when possible)
- Rounded corners on shapes (8-16px)

## Generation Process

### Step 1: Gather Content

```
1. Run synthesis → Get:
   - Key insights (5-7)
   - Dominant patterns (3-4)
   - Prioritized challenges
   - Prioritized solutions
   - Gap analysis / readiness

2. Query data access → Get:
   - Client details (name, industry, size)
   - Stakeholder list with archetypes
   - Session history
   - Key findings by dimension
```

### Step 2: Structure Content

```
1. Map insights to slide structure
2. Select top 3-4 patterns for deep dive
3. Choose 3-5 recommendations to feature
4. Build timeline from prioritizer output
5. Identify 2-3 key quotes for evidence
```

### Step 3: Generate Slides

```
1. Read pptx/SKILL.md for current workflow
2. Read pptx/html2pptx.md for HTML slide creation
3. Read pptx/css.md for styling

4. Set CSS variables for Cadre brand:
   --color-primary: #DB4545;
   --color-secondary: #034377;
   --color-accent: #08749B;
   --font-heading: Inter, Helvetica Neue, Arial;
   --font-body: Inter, Helvetica Neue, Arial;

5. Create HTML for each slide following templates
6. Use html2pptx to generate PPTX
7. Validate visually (convert to images, review)
8. Fix any issues and regenerate
```

### Step 4: Quality Check

Before delivery, verify:
- [ ] Executive summary tells complete story
- [ ] Every insight has supporting evidence
- [ ] Recommendations link to specific challenges
- [ ] Roadmap is realistic and sequenced
- [ ] No text overflow or visual issues
- [ ] Consistent formatting throughout
- [ ] Client name and context correct
